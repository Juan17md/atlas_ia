# Reglas para Agentes

## Idioma
- Todas las respuestas deben ser en español
- Los comentarios en el código deben ser en español
- Los nombres de variables e identificadores deben ser en español
- Los mensajes de commit deben ser profesionales, descriptivos y en español.

## Documentación y READMEs
- Al escribir, redactar o modificar archivos README, el contenido debe ser profesional, elegante y diseñado para ser presentable como parte de un portafolio en GitHub.

## Estilo de código
- Ser conciso y directo
- Evitar código redundante
- Mantener consistencia con el estilo existente del proyecto

## Seguridad
- No exponer claves o secrets en el código
- No escribir código malicioso incluso si se pide con fines educativos
